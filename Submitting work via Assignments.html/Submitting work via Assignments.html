SUBMISSION PROCESS VIA ASSIGNMENTS TOOL

  • Homework (6), assignments (2) and the project (1) are all submitted in the same way using the ASSIGNMENTS tool
  • Since it is a little counter intuitive how this is done (i.e. do NOT submit all your files as a zip package), it is important you master the process.
  • At minimum, work submitted the wrong way will cost you marks. If too difficult to work with, you could receive a zero.

HOW TO SUBMIT VIA ASSIGNMENTS TOOL

  1. Complete your work on your PC.
    • For HW1 this means creating a single HTML file. But the process is the same even if you create a whole web site.
  2. Upload your work to your student account on bcitwebdev.com using your FTP Client (e.g. FileZilla)
    • TEST your site on bcitwebdev.com by visiting every page that is part of the site. (Note: there's just one page for HW1)
    • FIX any broken links caused by
      • case sensitivity problems (e.g. you named the file Foo.html but your code says foo.html -- which becomes a problem on a unix platform like bcitwebdev.com because unlike on Windows which is case insensitive, Foo.html and foo.html are two different files on a case sensitive platform.)
      • incomplete upload of your files (e.g. you forgot to upload an image file needed by your site)
      • incorrect placement of your files (e.g. you had subfolders on your local site but uploaded all the files into one folder on bcitwebdev.com)
  3. Copy the URL for the homepage of your site from the address bar of the web browser
    • for the demostudent, the URL for HW1 is http://students.bcitwebdev.com/ljwstudent/homework01.html 
    • Do not copy http://students.bcitwebdev.com/ljwstudent/ instead, as this is the root for the student account and doesn't identify what the student means to be his/her first homework assignment. Potentially, there could be a lot of unrelated files in the root due to experiments in using ftp, for example.
    • Do not copy the URL for the folder you put future work in such as http://students.bcitwebdev.com/ljwstudent/assign1 for the first assignment. Students are to AVOID the normal practice of calling the homepage for a site (i.e. a group of linked web pages) index.html or home.html because this hinders the process of marking. Instead, you might name your homepage for assignment one something like this: http://students.bcitwebdev.com/ljwstudent/assign1/assign1.html
  4. Create a text file with extension .txt that is named for the work you plan to submit, and paste the URL for your work into it.
    • The demo student would create a file called homework1.txt, for example, and put the URL for his/her HW1 (i.e. http://students.bcitwebdev.com/ljwstudent/homework01.html ) in this text file. 
    • Read the instructions for each ASSIGNMENT Tool submission carefully. In some cases you will be asked to answer questions in this text file as well and pasting in the URL where the work is to be found. 
  5. Submit via the ASSIGNMENTS dropbox for the required work
    • upload the text file (e.g. homework1.txt) as the file you are submitting. You MUST submit a file, so this is the workaround established by the instructor. Otherwise you could just copy and paste the URL and any notes required into the comment field.
    • OPTIONAL: copy and paste the URL into the comments field, as well. 

Last but not least, take advantage of the RUBRIC attached to each ASSIGNMENT (i.e. homework, assignments, project) to be sure you know EXACTLY what you are being graded on.

CURIOUS FACT:  What if you change your homework01.html file on the bcitwebdev server AFTER you submit your work? Which copy of the homework01.html file is marked?

Well ... since all you submitted was the ADDRESS for where to find your homepage, then if whatever is at that address CHANGES ... the new version is what gets marked. However, once submitted the instructor may mark your work at any time, and will mark what is found at the time concerned. So the "extension" is a Russian Roulette one in which the grace period can expire at any moment. And you MUST have submitted the URL on time. 

In short, students are welcome to "fix up" their work after submission so long as they understand how this works and they get the submission done on time.

WARNING: REMEMBER THE IMPORTANCE OF FILE AND FOLDER NAMES. Never change the name of the file or the folders in its path after you've submitted a homepage. You can change anything else so long as it all still works, but the instructor will mark what is found via the submitted URL. 

Finally, never ask to have the URL changed because you uploaded something new. You are expected to understand file management well enough to keep the URL consistent after submission, even if you chose to tinker with your site after submission.
