Catalog navigation

The secondaru menu for the page is in the left column of the two column design. It follows the same design theme as the primary navigation, and I think is clear enough that it doesn't require a heading. It also inludes the "Back to Top" link in the bottom right.

I set the position of the side navigation menu to be fixed so that even if the user scrolls down the page it will still be visible. And the "Back to Top" link can be clicked from anywhere on the page.

The order of the side nav menu follows the order on the page, which is the beginner package first, and then the other packages in order of increasing cost.

DIV tag (With id attribute)

<div id="content"> ...  </div> 

This div tag contains the content of the introduction page, and was used to hold the text left to the image. Both the div with id="content" and the image are children of the main tag, which has overflow="auto" and allows for the formatting to work correctly

Find this div on the Introduction Page.
DIV tag (With class attribute)

<div class="employee-div"> ...  </div> 

<div class="employee-contact"> ...  </div> 

Both of these examples are used to format each employee. Divs with class="employee-div" hold all the information and picture of that employee, while the employee-contact class was needed to provide different formatting and hold all the contact information for each employee

Find this div on the Contact Page.
Semantic element + CSS example

I used the main tag as the container for all of the non-header site content- ie. all of the non-repeating site content.

I formatted the main tag with a max-width, overflow, set the margins, and gave a default background colour.

This tag was used on all of the pages, including this one. And the CSS for the main tag is in the style.css file
box model

The entire website uses the box-model. An example would be <div id="content"> ...  </div>  in the catalog page. The embedded CSS for content includes setting the margins and the padding so that it formats correctly with the side-nav and the top-nav.

Find this example of the box-model on the Catalog Page.
Two new HTML5 tags

Two HTML5 tags I used are the section tag and the header tag.

The section tag I used in the catalog page in-order to semantically seperate each of the packages and I used the header tag on every page for the company name and the main navigation

The section tag can be found on the Catalog Page. The header tag is on every page, including this one.
Cascade CSS

I overrode the external CSS for the main tag in the catalog page using embedded CSS because the changes were specific to that page. I set the max-width to be 100% instead of the external CSS which was 95%. I thought the style of the catalog page looked a lot better when it stretched across the entire browser.

I also used inline CSS to override the embedded CSS and external CSS for the h2 tag in the contact page. There was no reason to do this besides fullfilling the assignment requirements as I couldn't find a situtation during the assignment that warrented inline CSS.

Find this example of overriding the external CSS using embedded CSS on the Catalog Page.

Find this example of overriding both the external CSS and embedded CSS using inline CSS on the Contact Page.
